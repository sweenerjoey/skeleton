About Skeleton.CSS

Inspired by Dave Gamache's framework of the same name, Skeleton.CSS uses a selection of intuitive utility classes to provide basic functionality for CSS grids and flexboxes, and establishes a basic spacing scale. No visual characteristics are applied through Skeleton.CSS, allowing users to build their own type scales, buttons styles, or entire design systems on top of the basic framework.

CSS Grid

Any container can be assigned grid functionality using the class grid. The grid is mobile-first, starting at 4 columns and expanding to 12 at larger screen sizes. Various classes assigned in the HTML to direct children of the grid determine the item's placement, starting from mobile size up. All children of the grid must have at least one mobile class assigned, or it will default to spanning a single column at every breakpoint.

The grid classes follow a simple naming syntax. Most of the utility classes are aliased under various names, allowing you to use named values like "thirds" and "quarters" as well as numbers 1–12.

CSS Flexboxes

Add class flex to any container to enable flexbox, and then use additional classes to customize the flexbox's behavior. The same principles of class naming to grid applies here: specify things like flex-direction per breakpoint.
